HIGHVIBE SPECIFICATION v0.2.1:

.0 FORMAT
A .hvibe.plus file uses JavaScript object notation, not strict JSON nor JavaScript. Keys are never quoted. Backtick strings are LLM-direct, highest priority, interpret literally. Quoted strings are LLM-indirect, lower priority.

.1 TOP-LEVEL FIELDS
hvibe_version, app, description (LLM-direct), stack (LLM-direct, respect strictly), catalog (optional, inline or external path), units (each key is a unit name, contains features).

.2 CATALOG
globals: backtick field declaring shared constants and variables used across all features. Read first.
logic: arrow functions with JSDoc comments. Treat as reasoning scaffolds.
code: literal code snippets, use as-is only when explicitly marked as final.
tests: arrow functions asserting passing conditions. Must pass before finalizing. Keywords: MUST BE, MUST ALWAYS, MUST NEVER, OTHERWISE.
If catalog is a string, treat it as a path to an external file. Multiple catalogs complement each other.

If a logic or code entry is prefixed with `STRICT:`
- It becomes a deterministic algorithm specification.
- The LLM must implement it exactly as described.
- No generalization, abstraction, or architectural extension is allowed.
- The logic must be treated as executable intent, not interpretative guidance.
- Any deviation is considered an implementation error.

.3 UNITS
Each unit has an optional name, description (LLM-direct), references array, and a features object.

.4 FEATURE FIELDS
description (LLM-direct), inputs (LLM-direct), rules, tests (catalog dot-path refs, must pass before finalizing), blueprints (catalog dot-path refs or backtick instructions), depends_on (LLM-direct).

.5 FEATURE RULES
obligations: behaviors to implement.
constraints: hard limits to enforce.
Keywords: MUST BE, MUST ALWAYS, MUST NEVER, OTHERWISE.

.6 LOCK FILE
Format: { hvibe_lock_version, locked_at, features: { "unit.feature": { locked: true | false } } }
locked: true — feature is final, do not touch.
locked: false or absent — feature is pending generation.
If a locked feature must be modified, flag it and wait for user approval.
